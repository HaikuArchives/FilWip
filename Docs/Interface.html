  User Interface

The graphical user interface of FilWip is designed in such a way it can handle any number of plugins. It has a collapsable/expandable tree-view structures for hierarchical plugins. This makes organising plugins easier. Example, all items of NetPositive can be 'grouped' into one item called NetPositive and the like. FilWip also sports the toolbar found in its predecessor. Below is a screenshot of main window.

The main window consists of the toolbar, the various options (checkboxes), the clean-up button and the presets menu. When you fire up FilWip you will get a similar window. Some plugins may not be shown to you. For example, if you don't have Opera browser installed then the Opera plugin will not load. This is due to the "Check Install" behaviour of that Plugin. You can disable this behaviour (meaning all plugins will load regardless of what is installed in your system) here.

The various components of the main window, how they work are explained below...


  The Toolbar

Help - Opens this documentation in your browser
About - Shows copyright, license and developer information
Save Preset - Save the current options to a Preset file
Preferenecs - Edit FilWip's preferences
Preview - (Currently un-implemented)
Select All - Selects all the options
Deselect All - Un-checks all the options
Select as needed - only those options that have files to clean will be selected

The toolbar is quite straightforward and doesn't need much explaination. Just click them and stay happy :)

If you forget what toolbar button does what you can let the mouse hover over the button and you will get a small tip identifying the button. The tip also shows you any keyboard shortcut, if any, for example the Save Preset button has the Cmd + S shortcut.

The "Default" preset shown in the main window is actually FilWip's settings file displayed under that name. If you load other presets and you want to revert back to the old settings file (before you close FilWip) you can choose the Default preset. You can have other presets with the name Default, but the settings file preset will always be the first in the list so you can distinguish it from other presets named Default (if any).

You can read more about Presets here.


  Overview Information

The overview concept is a new feature in FilWip. As you would have seen in the main window each item shows information about what is there to be cleaned up. This is called overview, you simply get to see the number of files, total size of the files that you will recover when you delete them.

Each of these information labels are called Infostrings or Overview info. They can update "live" when changes occurs to files it monitors. Example, if Bash History file is removed FilWip will update itself (assuming you have not deleted the Bash History plugin). This is of course, if you have enabled Live monitoring in Preferences.

Notice how the Infostrings's displays "items" and not "files" or "folders". This is because when monitoring recursive plugins FilWip monitors both files and sub-folders. Thus it will not differentiate between files and folders and will simply tell them as items. The Report that you see after a clean-up will show the full details such as number of files, folders size etc. For non-recursive plugins, "n items" can be taken as "n files" as no sub-folders are monitored.

An advantage of having this monitoring is that you can double-click them to open the folder (through Tracker) they are monitoring and see a more detailed view of what files will be removed. If the item happens to be a single file, the parent folder will be opened.

Also please note that the Select as needed option (described above in the toolbar) works on the values of these Infostrings. Thus if you have Live monitoring turned off, Smart checking will not be shown for use.
