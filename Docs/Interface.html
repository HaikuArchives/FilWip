User Interface

The graphical user interface of FilWip is designed to handle any number of plugins. It has a collapsable/expandable tree-view structures for hierarchical plugins. This makes organising plugins easier. For example, all items of the operating system can be 'grouped' into one item called 'System logs' and the like. Below is a screenshot of the main window.

The main window consists of the toolbar, the various options (checkboxes), the "Clean up" button and the "Presets" menu. When you fire up FilWip, you will get a similar window, though some plugins may not be shown to you. For example, if you don't have QupZilla browser installed then the QupZilla plugin will not load. This is due to the "Check Install" behaviour of that Plugin.

The various components of the main window, how they work are explained below...


  The Toolbar

Select All - Selects all the options
Deselect All - Un-checks all the options
Select as needed - only those options that have files to clean will be selected
Save Preset - Save the current options to a Preset file
Preferenecs - Edit FilWip's preferences
Help - Opens this documentation in your browser

You'll find all these items in the menu bar as well.

The "Default" preset shown in the main window is actually FilWip's settings file displayed under that name. If you load other presets and you want to revert back to the old settings file (before you close FilWip) you can choose the Default preset. You can have other presets with the name Default, but the settings file preset will always be the first in the list so you can distinguish it from other presets named Default (if there are any).

You can read more about Presets here.
