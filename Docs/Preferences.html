  Preferences

FilWip is a tweakable utility. It has several options that I hope will make it flexible enough to suit all users. If you have any other options you would like to add, e-mail me about it. FilWip splits the set of options into several categories so it is easier to find the option you are looking for. They are clubbed into categories like "Report", "Plugins" etc. Here is a screenshot of the Preferences window:

Each option of each category is explained below in detail.


  Report

This category has report related options, ie you can set what you want to see after a clean-up operation is complete. If you turn OFF all these options, the report won't be shown to you after cleaning up.

Disk space freed
Default: ON
Toggle showing the storage space you recovered after a clean-up.

Number of files, folders deleted
Default: ON
Toggle showing the number of files and folders that were deleted in a clean-up.

Time taken to clean-up
Default: ON
Toggle showing the time taken to complete the clean-up operation.


  Items

This category has options that pertain to each plugin-item.

Check items at startup
Default: OFF
When FilWip starts if this option is set, if an item has something to clean it is selected, otherwise it is deselected. This will happen only once at startup.

Check items live (if monitoring)
Default: OFF
When this option is set, and if Live monitoring option is also set, as and when some files are there to be cleaned the option will be selected/deselected accordingly.

Warning!!! This option can be dangerous if you want to delete only some selected items. Unless you always clean all the plugin-items, setting this option is not recommended.

Uncheck items after clean-up
Default: OFF
When this option is set, after a clean-up operation all the plugin-items are deselected (and if you close FilWip it will save all items as deselected).


  Loopers

Loopers are program elements (threads) that maintain a list of files for each plugin-item. Each plugin-item spawns a new looper. Loopers are responsible for monitoring files, folders, preparing the list of files of to delete. The "Eraser" looper is a special looper that does just the deletion. There is only one eraser looper. All other loopers are specific to each plugin-item. FilWip offers some options to fine-tune the loopers to suit your needs.

Make loopers "sync" to disk
Default: OFF
When this option is set, after a clean-up operation the loopers will run a "sync" command to commit all changed blocks to disk. This is not required anymore as FilWip uses a high level delete which doesn't need sync to be executed. In case your BeOS volumes do not update their sizes (free space or used space) after a clean-up keep this option ON and it should be fixed.

SafeCheck
Default: ON (disabled)
This option ensures your plugins never delete files or folders critical to the system. This includes home folder, beos folder (and things inside beos folder) and the like. Warning: There are still some files critical to the system that plugins can delete like things inside settings folder in home etc. these aren't checked. Only the following paths are checked:

	/
	/boot
	/boot/home
	/boot/home/config
	/boot/home/config/settings
	/boot/home/Desktop
	/boot/beos (full protection)

This option has not been made available to change. It will always stay ON.

Live Monitoring
 Default: ON
Keeping this option ON will mean all files/folders targetted by plugin-items will be monitored for any changes. For example, when you have FilWip open and some background app creates/deletes some temporary files they will show up from FilWip live - as and when they change. This will help you keep track of what is where.

Eraser priority
Default: n/a
Eraser priority is the priority you want to give the eraser looper. A higher priority will mean quicker clean-ups but not all the time. It can also mean interrupting more critical tasks like audio playing which "really" requires high priority. A priority of "Normal [10]" is ideal, but you can change it as needed. But remember, having a higher priority doesn't always guarantee faster clean ups.

Port capacity
Default: n/a
An important setting. This setting will determine how capable are loopers to handle massive file changes. A large port capacity will mean when many files change at once the notification queue for each looper won't get full easily. I don't want to get too technical but remember a higher port capacity ensures file changes are "caught" properly. A lower capacity might "drop" some changes. This can lead to outdated information displayed to you.

If you have Live Monitoring turned ON and you want to monitor files accurately, I suggest a port capacity of atleast 1000. Higher the better, but more memory will be used (but might not make much of a difference in high-end systems).


  Remember

These options will be remembered and restored when FilWip starts.

Tree state
Default: ON
When this option is set, the tree-view structure is preserved (hierarchial plugins expanded or collapsed state) accross app restarts.

Window position and sizes
Default: ON
When this option is set, the position and (if appropriate) size of windows in FilWip are remembered and restored. This way you need not adjust the position/size of windows everytime you run FilWip.

Clean-up items (changeable by Item options)
Default: ON
When this option is set, the plugin-item values (selected or deselected) is saved on quitting FilWip and restored on startup. After starting-up with these option they can be changed by the options set in Items category.


  Plugins

This category allows you to adjust various settings when plugins are loaded. Because plugins are loaded while starting up, any changes to these option will take effect ONLY after you close and open FilWip again i.e. after restart the app. Remember this point. So don't change some critical option here and think that it has been applied immediately.

Load plugins asynchronously
Default: OFF
When this option is set, FilWip is first launched. After the window is responsive plugins load one by one. Thus on slow systems it will show the window quickly then do the loading which may take some time. When this option is not set, all plugins are fully loaded and only then is the window shown to you. On fast systems (~ > 500 Mhz) you can keep this setting off.

Debug mode
Default: OFF
When this option is set, FilWip gives you details of how each plugin loads, one by one. For each plugin being loaded you get a window with the details and the validity of each item. This maybe useful to find out any faulty plugin. Otherwise if a plugin is faulty, FilWip will simply give an error and quit. This way you can find out exactly which plugin(s) cause problems to FilWip.

By default it is OFF meaning all plugins are correct. Before you add/edit plugins run FilWip and set this ON. Once you get it to work you can turn it back OFF again.

Check install path existence
 Default: ON
When this option is set, the CHECK INSTALL OF for plugins that specify one are verified before loading that plugin.

For instance, the Opera plugin has a CHECK INSTALL OF value as /boot/home/config/settings/Opera. The existance of this path is verified before loading the plugin. Thus if you don't have Opera installed you will not have the aforementioned path, thus the plugin will not be loaded.

Allow recursion into sub-directories
 Default: ON
When this option is set, plugins (that specify recursion) are allowed to recurse into sub-directories. For example, the Trash plugin wants recursion, so that everything inside Trash is deleted even sub-folders and its contents. But, you can control that setting universally across plugins from here.


  Miscellaneous

Other miscellaneous options are available here.

Confirm before clean-up
 Default: ON
When this option is set, you are asked to confirmation after you choose to clean-up.

Register file-types are startup
Default: OFF
Sometimes when Preset files aren't recognized by your system you may need to run FilWip, turn this option ON and re-run FilWip to register the Preset file type. By default it is OFF. Once you have registered the Preset file-type you can keep this option OFF.

Quit after clean-up
 Default: OFF
When this option is set, after a clean-up operation, FilWip will quit itself.
