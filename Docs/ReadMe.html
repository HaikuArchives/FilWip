This introduction section is for you newbies or the type who actually read documentation before trying out the app.

FilWip is a disk maintainance utility. Its a small program that helps you keep your hard disk clean from unwanted data, such as temporary files, logs, caches etc. that accumulate usually in the background, and if you are a casual user you might not be aware of them.

Temporary files, system event logs, caches etc. occupy some considerable disk space. Most of these files get automatically created and get created quite often. Cleaning all these items manually, traversing from folder to folder, can be a difficult, tedious and time consuming task.

FilWip makes this job as easy as possible implementing a graphical user interface. FilWip is a plugin-based app, meaning it's fully customizable and extensible.

User Interface

The graphical user interface of FilWip is designed to handle any number of plugins. It has a collapsable/expandable tree-view structures for hierarchical plugins. This makes organising plugins easier. For example, all items of the operating system can be 'grouped' into one item called 'System logs' and the like. Below is a screenshot of the main window.

The main window consists of the toolbar, the various options (checkboxes), the "Clean up" button and the "Presets" menu. When you fire up FilWip, you will get a similar window, though some plugins may not be shown to you. For example, if you don't have the QupZilla browser installed then the QupZilla plugin won't get loaded. This is due to the "Check Install" behaviour of that plugin.

The various components of the main window, how they work are explained below...

The Toolbar

Select all - Selects all the options
Deselect all - Un-checks all the options
Select as needed - only those options that have files to clean will be selected
Save preset - Save the current options to a Preset file
Preferenecs - Edit FilWip's preferences
Help - Opens this documentation in your browser

You'll find all these items in the menu bar as well.

The "Default" preset shown in the main window is actually FilWip's settings file displayed under that name. If you load other presets and you want to revert back to the old settings file (before you close FilWip) you can choose the Default preset. You can have other presets with the name Default, but the settings file preset will always be the first in the list so you can distinguish it from other presets named Default (if there are any).

You can read more about Presets here.

Usage

Using FilWip is straight forward and simple.

As you might have seen in the User Interface section, the main window loads all available plugins. If you don't want to remove particular items, un-check their checkboxes.

Once you have selected the items you need to remove, click on Clean up. You will have to confirm that decision before the system is cleaned up. This confirmation can be disabled in the Preferences.

The Progressbar

The clean-up process will begin after you clicked Clean up in the confirmation dialog. A progress bar informs of the cleaning process. When finished, a report will detail the number of files cleaned, space recovered etc.

When finished, a Report summarized how many files/folders were deleted, how much disk space this freed, and how long it took. You can turn off the Report in the Preferences. You can make FilWip quit automatically after this operation there, too.

FilWip can be closed while the clean-up is taking place. The delete process will then be paused and you'll be asked for confirmation to force the quitting. Clicking Don't stop will resume the delete process, Stop will cancel the delete process and quit FilWip.

Before using FilWip to clean up your system, close all interactive programs that are running. This isn't necessary 99.99% of the time, but in a rare situation it can cause problems for other running applications. For example, while compiling a project, many temporary files are created and deleted. FilWip interfering here, can cause crashes and other problems.
This is just a generic warning. So far, it hasn't caused any problems for me and I haven't received any such reports from users. But as they say, better be safe than sorry and finish all serious work before invoking FilWip...

From Terminal

FilWip offers a neat way to make it act like a simple script program. It can come in handy, not having to open FilWip everytime you want to do a cleaning-up. A simple "silent" mode allows you to do this. Command-line parameters are explained below.

Parameter Description
-preset <preset_file> Loads the items from the specified the Preset file and launches the graphical user interface (if necessary)
-silent Begins the clean-up process in silent mode. In silent mode, the graphical user interface is not launched, graphical user pop-ups/errors or report are not shown. Thery are shown in text mode instead.

Please note that both the above options can be combined, but only in the order specified i.e. -preset first, -silent next. This means you can load any Preset file and clean up in silent mode.

Specifying only a Preset file (without the -silent option) at the command-line will launch the graphical user interface and the specified Preset (if it exists) will be loaded and ready. Note that you must specify the full path (or relative path) to the Preset file. FilWip will not check the list of Presets in the Presets folder for you.

Here is a sample report from the command-line:

Report

  Items erased:
          12 folders
          312 files
  Time taken:
          3.007536 seconds

Plugins

Plugins are what makes FilWip a versatile and flexible app. Plugins (as far as FilWip is concerned) are a way of specifying what files to monitor and delete. They are simple text files, written in a particular format understandable by FilWip, that specify the locations to clean.

You get a set of standard plugins with FilWip. You can edit these plugins to suit your system, but mostly you won't need to. You'll find the plugins at ~/config/settings/FilWip/Plugins/. Some of the standard plugins include: WebPositive files, Custom Query files, Temporary files, Bash history file etc.

Before we proceed explaining the format specs for the plugin, here are two sample plugins. Just read through them once and read the explanation below.

Here is a hierarchical plugin - meaning you can "expand" the plugin and it will have sub-items. Here is how the WebPositive plugin looks: 

;; WebPositive plugin

[WebPositive]
TYPE=hierarchical
CHECK INSTALL OF=/boot/home/config/settings/WebPositive

[Cache]
FOLDER=/boot/home/config/settings/WebPositive/NetCache
EXCLUDE FILE=Cookies
RECURSE=

[Cookies]
FILE=/boot/home/config/settings/WebPositive/Cookies
EXCLUDE FILE=
RECURSE=

[History]
FILE=/boot/home/config/settings/WebPositive/BrowsingHistory
EXCLUDE FILE=
RECURSE=

[Site icons]
FILE=/boot/home/config/settings/WebPositive/WebpageIcons.db
EXCLUDE FILE=
RECURSE=

Here is a linear plugin - meaning it does not have any sub-items and can clean only one location. Here is how the Custom queries plugin looks: 

;; Custom queries plugin

[Custom queries]
TYPE=linear
CHECK INSTALL OF=

[Custom queries]
FOLDER=/boot/home/queries
INCLUDE MIME=application/x-vnd.Be-query
RECURSE=

Syntax and Keys

TYPE (Required) The type of plugin, whether linear or hierarchial
CHECK INSTALL OF (Optional) This path must exist, or else the plugin will not be loaded
FOLDER (N/A) The location to delete. Either FILE=<file> or FOLDER=<dir>
EXCLUDE FILE (Optional) A file to exclude in the folder being targetted. EXCLUDE FILE=<filename>
INCLUDE MIME (Optional) Files of this type only are touched, INCLUDE MIME=<mime-type>
RECURSE (Optional) Whether FilWip must touch sub-folders and files.
;; (N/A) Comments - these are skipped
1st [Custom queries] (Required) Plugin name used in Presets internally, not shown otherwise
2nd [Custom queries] (Required) A description. Shown in the checkboxes/treeview.

Let's go from top to bottom, explaining each unique line in the plugin format:

;; Comment!
Lines beginning with a double semicolon are considered comments and are skipped. You can use them only at the beginning of a plugin i.e., until the name of the plugin is specified.

[Name]
Lines enclosed within "[" and "]" are considered to contain names for each item. In case the TYPE of the plugin is hierarchical, this name will be used as label for the expandable treeview. This is what you will see in FilWip for hierarchical plugins. For linear items this name will be displayed. The next name section following it will not be used.

TYPE=linear
TYPE=hierarchical
A very important clause. This clause specifies if the plugin has sub-items or not. TYPE must be either "linear" or "hierarchical". Careful you don't misspell it and you cannot leave it blank.

CHECK INSTALL OF=[<path>]
This clause allows you to stop loading the plugin if a certain file or folder does not exist. For example, the QupZilla plugin checks for a path (in the settings folder) to confirm if QupZilla really exists. If it doesn't, the plugin will not be loaded. You can leave this blank, meaning no checks are performed.
For example, Trash has a blank field here since Haiku is assumed to have a Trash location.

FOLDER=<path>
FILE=<path>
This clause specifies either a folder or file to clean. The path must be the full absolute path. A safety check is performed so that no system critical folders/files are destroyed. For example, you can't specify /boot/home as it is critical to the system.

Also, you cannot specify both FOLDER and FILE clauses for the same plugin item. It's either FILE or FOLDER. See the example plugins above.

EXCLUDE FILE=[<filename>]
INCLUDE MIME=[<mime-type>]
The EXCLUDE FILE clause allows you to exclude a file (by name) from the list of files in a folder. This will apply only if a folder is specified. A blank field is allowed meaning no files are left out.

The INCLUDE MIME clause specifies that only files with the specified mime-type (like "application/x-vnd.Be-query") should be deleted. For example, if you want to clean only .html from a folder but no other file-types like .gif etc., you can INCLUDE MIME=text/x-html. Blank field here is allowed meaning file-types aren't checked for the given scope of files.

You cannot specify both EXCLUDE FILE and INCLUDE MIME in the same plugin item.

RECURSE=[<yes>]
The RECURSE clause allows folder entries to recursively target all sub-folders. To delete all sub-folders and files, you must specify RECURSE=yes. Leaving it blank specifies that no recursion will take place, meaning sub-folders (and files within sub-folders) will not be touched. Obviously, this applies only to plugin-items that target a folder.

More info

Plugins as of now are 'delicate' structures in FilWip, error detection is minimal. So please make sure you write/modify your plugins very, very carefully. Make sure you don't leave extra line-breaks, don't leave extra spaces, don't misspell words. The best way to go about writing a plugin is to copy and paste an existing plugin to a file and edit and modify it carefully.

Plugins are not case-sensitive, i.e. "FOLDER=" is same as "folder=". I wrote them in capitals as it looks much neater and error detection is easier. But remember, the paths in Haiku ARE case-sensitive, e.g.: /boot/home/Programs is not the same as /boot/home/programs.

Preferences

FilWip splits the set of options into three categories:

At start-up

Live monitoring
This means all files/folders targetted by the plugins will be monitored for any change.
For example, when you have FilWip open and some background app creates/deletes some temporary files they will show up in FilWip live - as and when they change.

Select as needed
If a plugin has anything to clean, it is selected, otherwise it is deselected. This will happen only once at startup.

Before clean up

Ask confirmation
You'll be asked for confirmation after choosing to clean up.

After clean up

Show report
Will show a report detailing the number of files deleted, the gained disk space etc.

Deselect all
All plugin-items will be deselected (and if you close FilWip it will save all items as deselected).

Quit
FilWip will automatically quit after the clean up action has finished.

Presets

A preset is simply all your clean-up options saved in a file. Thus a preset is a file that contains key/value pairs regarding what options are selected. When you double-click such a file from Tracker, FilWip will load all the settings from that file to the main window, ready for you to click Clean up.

This is the primary and probably the only use of presets in FilWip. To load a particular set of options for quick loading and cleaning, saving you the time of re-selecting the options each time.

You can create several presets for different needs, like a preset called "Net options" that will have all internet based cache options selected.

To save a preset, open FilWip and choose the options you want to save. Then click the Save icon from the toolbar. By default you will be prompted to save the preset into the Presets folder in ~/config/settings/FilWip/, but you can save it anywhere. The advantage of saving the preset in the Presets folder is that when FilWip loads, all presets in the folder will be added to the Presets menu (like shown in the image above), so you can quickly open it without having to navigate to the folder where you saved it in.

The Default preset shown is actually FilWip's settings file displayed under that name. If you load other presets and you want to revert back to the old settings file (before you close FilWip) you can choose the Default preset. You can have other presets with the name Default, but the settings file preset will always be the first in the list so you can distinguish it from other presets named Default (if there are any).

Presets can also be loaded from the command-line, see topic on using FilWip From Terminal.
