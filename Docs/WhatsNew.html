  What's New

This is the first public release of FilWip and since it is meant to be a replacement of the BeOS Clean-Up Manager it has the following differences - improvements and exclusions in comparison to its predecessor:

  • Plugins — User-editable files that allow you to control what FilWip cleans.
  • Progress bar — Now FilWip shows the progress of the clean-up operation.
  • Live monitoring — FilWip monitors changes to files it is supposed to delete "live".
  • Overview — FilWip shows overview of files - number of files, sizes
  • Auto selection — FilWip has several options for automatic selection of plugins.
  • Smart loading — FilWip can load only those plugins that are appropriate to your system.
  • Safety Check — Ensures your plugins don't delete files critical to your system.
  • Recursive deletion — Meaning, files inside sub-folders are also deleted.

These are just the features that FilWip has that its predecessor app did not have. Its full list of features are many more. FilWip is written in a much more efficient manner than BeOS Clean-Up Manager. Though the latest version cleaned up code, FilWip's code is more clean and efficient. FilWip fixes several problems through means of Plugins. Now all you people that wanted Mozilla support can have it in the form of a Plugin.

Some other features include:

  • ... completely generic, no paths hardcoded

The downside of FilWip? Yes it has one demerit in comparison to BeOS Clean-Up Manager as far as I know: It doesn't have Preview yet. I felt this isn't necessary anymore because Preview was needed by people who wanted to see what files will be deleted. Now that they have Overview infostrings for this, I don't see a point in having a seperate Preview. Just seems like more pointless coding to me. Anyway, if many of you still want preview write to me, and if there is sufficient interest I will code it.

FilWip in terms of pure performance will be a touch slower than its predecessor. This is due to several reasons. First, because FilWip implements plugins, thus it has the ability to "filter" certain files while deleting. So checks are made for this for each file (or folder). Secondly, FilWip shows the progress bar while cleaning up - meaning updating the progress bar will take up CPU. Thirdly, FilWip counts all the files and folders before deleting (to show the progress of the whole operation) which its predecessor did not do.

But if you have a decent system, say ~600 MHz or higher, the performance will be very negligible. For in my 450MHz AMD K6-2 the performance was only about 0.5 to 1.2 seconds. As system speeds increase that will reduce drastically. Another advantage of FilWip is that it uses lesser memory despite the fact it spawns more than double the amount of threads BeOS Clean-Up Manager uses.
